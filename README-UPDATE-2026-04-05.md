# 2026-04-05 Update

本次更新重点修复了国外期刊返回空结果的问题，并继续推进新增来源的协议化检索。

## 核心变化

- `GEOPHYSICS / Petrophysics / OnePetro`
  - 搜索阶段改成对 `Crossref` 和 `OpenAlex` 做容错聚合。
  - 即使其中一个上游暂时失败，另一个上游仍然可以返回结果。
  - 搜索缓存已升级版本，旧的空缓存不会继续污染新结果。

- `CQVIP / 维普`
  - 已改成纯协议搜索。
  - 服务会先读取首页 SSR 参数，再按站点前端相同的签名规则构造请求头。
  - 日常搜索不再需要弹出可见浏览器。

- `万方`
  - 已改成 `protocol-first`。
  - 先尝试直连公共检索页。
  - 如果公共结果页只返回未注水 shell，再自动回退到浏览器渲染兜底，保证结果可用。

- MCP 输出
  - 现在会自动移除大体积 `raw` 字段。
  - 这可以避免国外元数据源把 `OpenAlex` 的超大原始对象直接塞进工具返回，减轻 Chatbox 等客户端的响应压力。

## 对使用方式的影响

- `search_geophysics`
  - 现在应当恢复正常返回结果。

- `search_vip`
  - 现在默认走纯协议，不再依赖可见浏览器。

- `search_wanfang`
  - 现在优先走协议；只有站点返回未渲染结果时才会回退浏览器。

## 建议

更新代码后请至少执行一次：

```powershell
npm.cmd run build
```

然后重启你的 MCP 客户端，让客户端重新加载新版本服务。
